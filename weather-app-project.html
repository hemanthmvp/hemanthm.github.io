02

What I Built — All Components

Custom Table
u_weather_app
Built a new ServiceNow table from scratch to store weather records. Includes 12 custom fields: temperature, humidity, condition, UV index, wind speed, air quality PM2.5, lat/lon, timezone, and more.
Script Include (Server-Side)
WeatherAPIHelper
Extends AbstractAjaxProcessor to handle GlideAjax calls. Reads the API key from gs.getProperty() (secure sys property), calls RESTMessageV2, parses the JSON, and returns all weather fields.
UI Action (Client-Side)
Get Weather Button
Validates the city field, shows an info message, creates a GlideAjax call with sysparm_city parameter, and in the callback populates all 13 form fields using g_form.setValue().
Client Script
Hide Weather Details onLoad
On new records, hides the weather_details form section. On existing records, checks if city is populated — shows the section if yes, hides it if no. Clean UX with zero flicker.
REST Message + HTTP Method
Weather API · Get_Current_Weather
Configured outbound REST Message pointing to api.weatherapi.com/v1/current.json with variable substitution for api_key and city_name parameters.
Security (ACL + Roles)
u_weather_app_user Role
Created a custom role u_weather_app_user with ACL rules to control read/write access to the weather table. Users without the role cannot access weather records.
Sys Property
weather_api_key
API key stored as a System Property (gs.getProperty('weather_api_key')) — not hard-coded in the script. Follows ServiceNow security best practices for credential management.
Application Menu + Modules
Weather App Navigation
Created a custom Application Menu "WEA" with navigation modules for the Weather App table. Users can access it from the ServiceNow left navigation without needing direct URL access.
Form Layout + Sections
weather_details Section
Designed a two-section form layout: input section (city name + Get Weather button) and a hidden weather_details section that only appears after a successful API call.

WeatherAPIHelper.js · Script Include · Server-Side
// Script Include: WeatherAPIHelper
// Extends AbstractAjaxProcessor to handle GlideAjax calls from client side
// Table: u_weather_app | Scope: Global

var WeatherAPIHelper = Class.create();

WeatherAPIHelper.prototype = Object.extendsObject(AbstractAjaxProcessor, {

    getWeatherData: function() {
        // Get city name passed from client side via GlideAjax parameter
        var cityName = this.getParameter('sysparm_city');

        // Fetch API key securely from System Properties (NOT hardcoded)
        var apiKey = gs.getProperty('weather_api_key');

        var result = {};

        try {
            // Reference the REST Message and HTTP Method configured in ServiceNow
            var rm = new sn_ws.RESTMessageV2('Weather API', 'get_current_weather');

            // Inject variable substitutions into the REST endpoint URL
            rm.setStringParameterNoEscape('api_key', apiKey);
            rm.setStringParameterNoEscape('city', cityName);

            // Execute the outbound REST call
            var response = rm.execute();
            var responseBody = response.getBody();
            var httpStatus = response.getStatusCode();

            if (httpStatus == 200) {
                var weatherData = JSON.parse(responseBody);

                // Map API response fields to result object
                result.status     = "success";
                result.city       = weatherData.location.name;
                result.time_zone  = weatherData.location.tz_id;
                result.time       = weatherData.location.localtime;
                result.country    = weatherData.location.country;
                result.region     = weatherData.location.region;
                result.temp       = weatherData.current.temp_c + "°C";
                result.humidity   = weatherData.current.humidity + "%";
                result.condition  = weatherData.current.condition.text;
                result.pm25       = weatherData.current.air_quality.pm2_5;
                result.uv         = weatherData.current.uv;
                result.feelslike  = weatherData.current.feelslike_c;
                result.wind_speed = weatherData.current.wind_kph;
                result.lat_long  = weatherData.location.lat
                                  + " / " + weatherData.location.lon;
            } else {
                result.status  = "error";
                result.message = "HTTP Status: " + httpStatus;
            }
        } catch (ex) {
            result.status  = "error";
            result.message = ex.message;
        }

        // Return JSON string back to GlideAjax callback
        return JSON.stringify(result);
    },

    type: 'WeatherAPIHelper'
});
GetWeather.js · UI Action · Client-Side
// UI Action: "Get Weather" Button
// Runs on Client Side | Table: u_weather_app
// Triggered when user clicks "Get Weather" on the form

function fetchWeather() {
    // Step 1: Get the city name from the form field
    var cityName = g_form.getValue('u_city');

    // Step 2: Validate — don't proceed if city is empty
    if (!cityName || cityName.trim() == "") {
        alert("Please enter a city name first");
        return;
    }

    // Step 3: Show loading message to user
    g_form.addInfoMessage("Fetching weather data for "
        + cityName + "... please wait.");

    // Step 4: Create GlideAjax call to WeatherAPIHelper Script Include
    var ga = new GlideAjax('WeatherAPIHelper');
    ga.addParam('sysparm_name', 'getWeatherData'); // method to call
    ga.addParam('sysparm_city', cityName);        // pass city to server

    // Step 5: Execute async — callback fires when server responds
    ga.getXML(weatherCallback);
}

function weatherCallback(response) {
    // Step 6: Extract the JSON string returned by the Script Include
    var answer = response.responseXML
        .documentElement.getAttribute("answer");

    var weatherData = JSON.parse(answer);

    g_form.clearMessages();

    if (weatherData.status == "success") {
        // Step 7: Populate all form fields with weather data
        g_form.setValue('u_city',           weatherData.city);
        g_form.setValue('u_time_zone',      weatherData.time_zone);
        g_form.setValue('u_local_date_time',weatherData.time);
        g_form.setValue('u_country',        weatherData.country);
        g_form.setValue('u_region',         weatherData.region);
        g_form.setValue('u_temperature_c',  weatherData.temp);
        g_form.setValue('u_fells_like_c',   weatherData.feelslike);
        g_form.setValue('u_humidity',       weatherData.humidity);
        g_form.setValue('u_weather_condition',weatherData.condition);
        g_form.setValue('u_air_quality_pm_2_5',weatherData.pm25);
        g_form.setValue('u_lat_long',       weatherData.lat_long);
        g_form.setValue('u_uv_index',       weatherData.uv);
        g_form.setValue('u_wind_speed_kph', weatherData.wind_speed);

        // Step 8: Reveal the hidden weather_details section
        g_form.setSectionDisplay('weather_details', true);

        g_form.addInfoMessage("Weather data loaded for "
            + weatherData.city + " successfully.");
    } else {
        g_form.addErrorMessage("Error fetching weather: "
            + weatherData.message);
    }
}
HideWeatherDetails.js · Client Script · onLoad
// Client Script: Hide Weather Details onLoad
// Type: onLoad | Table: u_weather_app
// Hides the weather_details section until data is fetched

function onLoad() {

    if (g_form.isNewRecord()) {
        // New record: hide weather section — user hasn't fetched yet
        g_form.setSectionDisplay('weather_details', false);

    } else {
        // Existing record: show section only if city data is present
        var city = g_form.getValue('u_city');

        if (city) {
            g_form.setSectionDisplay('weather_details', true);
        } else {
            g_form.setSectionDisplay('weather_details', false);
        }
    }
}

Key Learnings & Best Practices Applied

🔐 Security
API key stored as gs.getProperty('weather_api_key') — not hard-coded in the script. In production, use OAuth 2.0 credential store via Connection & Credential Aliases.
⚡ Async Pattern
Used GlideAjax (not synchronous Ajax) to avoid blocking the browser while the REST call executes on the server. This is the ServiceNow-recommended pattern for client-server communication.
🎨 UX Design
The weather_details form section is hidden on load using a Client Script and only revealed after a successful API call — providing a clean, guided user experience without page reload.
📦 Update Set
The entire application is packaged as a Remote Update Set XML — ready to deploy on any ServiceNow instance. This is the real-world method used to promote changes between dev → test → production.
