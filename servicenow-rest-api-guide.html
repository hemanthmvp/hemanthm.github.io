01   Core REST API Concepts

Retrieve Data
Fetch records from ServiceNow
POST
Send Data
Create records or send payloads
PUT
Update Data
Modify existing records
DELETE
Remove Data
Delete records from a system

📦
JSON
The data format used to exchange info
📡
HTTP Status Codes
Response indicators — tells you if the API call succeeded or failed (200 = OK, 401 = Unauthorized, etc.)
02   Two Types of REST Integration
① Inbound
External → ServiceNow
An outside system sends data INTO ServiceNow. ServiceNow acts as the receiver.
Jira creates an Incident in ServiceNow
② Outbound
ServiceNow → External
ServiceNow sends data OUT to another system. ServiceNow acts as the caller.
ServiceNow sends an alert to Slack

💡 Our use case is Outbound — ServiceNow calls the Slack Webhook.

03   Step-by-Step Implementation
1
Create a Slack Incoming Webhook
Get the Slack URL that ServiceNow will POST messages to.
How to do it
  • Go to api.slack.com/apps → Click "Create New App"
  • Choose "From Scratch" → give it a name (e.g., SN-Alerts)
  • Under "Add Features", click "Incoming Webhooks"
  • Toggle ON → Click "Add New Webhook to Workspace"
  • Pick the Slack channel → Click Allow
  • Copy the Webhook URL — it looks like:
    https://hooks.slack.com/services/T.../B.../xxx
2
Configure a REST Message in ServiceNow
Tell ServiceNow about the external system it will call — this is called a "REST Message".
How to do it
  • In ServiceNow, navigate to System Web Services → Outbound → REST Message
  • Click New
  • Name: e.g., Slack_P1_Alert
  • Endpoint: Paste your Slack Webhook URL here
  • Authentication: No authentication (Slack webhook handles it via the URL)
  • Click Submit
3
Add an HTTP POST Method
Define the actual POST call and what JSON payload (message) to send to Slack.
How to do it
  • Open your REST Message Slack_P1_Alert
  • Scroll to the HTTP Methods related list at the bottom
  • Click New → Name it post_alert
  • HTTP Method: POST
  • In the Content tab, set Content-Type: application/json
  • In the HTTP Request Body, paste this JSON:
JSON · Request Body
{
  "text": "🚨 *P1 Incident Created!*\nNumber: ${incident_number}\nDescription: ${short_description}\nAssigned To: ${assigned_to}"
}
About Variables
  • ${incident_number} — these are substitution variables
  • Their values will be injected dynamically by the Business Rule script
  • Click Submit to save the method
4
Create a Business Rule to Trigger the API Call
Write the server-side script that fires when a P1 Incident is created and sends the Slack alert.
How to set up the Business Rule
  • Go to System Definition → Business Rules → Click New
  • Name: Send Slack Alert on P1 Incident
  • Table: Incident [incident]
  • When: after | Insert: ✅ checked
  • Filter Condition: Priority is 1 - Critical
  • Check "Advanced" to unlock the Script tab
JavaScript · Business Rule Script
// Business Rule: Send Slack Alert on P1 Incident
// Table: incident | When: after | Insert: true
// Condition: Priority is 1 - Critical

(function executeRule(current, previous) {

  try {
    // Step 1: Reference the REST Message and its HTTP Method
    var rm = new sn_ws.RESTMessageV2(
      'Slack_P1_Alert',   // REST Message Name
      'post_alert'         // HTTP Method Name
    );

    // Step 2: Inject variable values from the current Incident record
    rm.setStringParameterNoEscape(
      'incident_number',
      current.number.getDisplayValue()
    );
    rm.setStringParameterNoEscape(
      'short_description',
      current.short_description.getDisplayValue()
    );
    rm.setStringParameterNoEscape(
      'assigned_to',
      current.assigned_to.getDisplayValue()
    );

    // Step 3: Execute the REST call
    var response = rm.execute();

    // Step 4: Log the response for debugging
    var statusCode = response.getStatusCode();
    gs.info('Slack Alert Sent. HTTP Status: ' + statusCode);

    if (statusCode != 200) {
      gs.error('Slack Alert Failed. Body: ' + response.getBody());
    }

  } catch (ex) {
    gs.error('Slack REST call exception: ' + ex.message);
  }

})(current, previous);
5
Test It — Send Instant Alert to Your Response Team
Create a P1 Incident and verify the Slack message arrives.
How to test
  • Go to Incident → Create New
  • Set Priority to 1 - Critical
  • Fill in Short Description and Assigned To → Submit
  • Check your Slack channel — alert should appear within seconds!
  • To debug: Go to System Log → All → filter by Slack Alert
  • Also test via REST Message → open post_alert method → click "Test"

05   Troubleshooting Checklist
🔌
Endpoint ConnectivityCan ServiceNow reach the Slack URL? Try the "Test" button on the REST method first.
🔑
Auth / Token ValidityIf using OAuth or API keys, check expiry. Most Slack webhook failures = expired or wrong URL.
📄
Request Payload StructureEnsure JSON is valid. A missing comma or wrong key name causes a 400 Bad Request.
📊
HTTP Response CodesLog getStatusCode() and getBody() — they tell you exactly what went wrong.
📋
Error LogsCheck System Log → All and filter by your script name for gs.error() output.
⚙️
Transform Scripts / Null ValuesAlways null-check fields before sending. Null in payload → 400 error from external system.
👆
Key Lesson from Real Projects Most integration failures are NOT caused by the script itself — they're caused by payload structure changes or auth token expiry on the external side. Always check those first before debugging your code.
